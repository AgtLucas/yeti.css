Typography

Font families used in majority of &yet's apps are Gotham and Helvetica Neue. When using Gotham be aware of the fact that it's web font served through HFJ Cloud Typography.

To use Gotham update your body font family accordingly and set up a new development project in Typography.com dashboard.

Headings and sizes

Headings run from 1—4. Base font size is the equivalent of 16px with em as a primary unit.

Disabled buttons

Being able to disable a button might come quite handy. Works only with button elements that have disabled attribute.

Button groups

You can group buttons by using .button-group class. For outlined versions use .outlined class along with the color denotation – .primary or .secondary – if you would like to change the default appearance.

Forms

Inputs and textareas

All elements have predefined :focus state and placeholders.

search

Validity

You can keep inputs in .input-wrapper (recommended). And add validity classes of valid and invalid which will change the styles of labels, inputs and icons (if present) accordingly.

If you are willing to use icons make sure the wrapper element also has a class .has-icon.Then add appropriate icons with this markup: i.ss-icon.ss-standard.ss-nameoftheicon

Other form elements

All basic elements have been covered: file inputs, checkboxes, radio buttons and selects. Despite checkboxes and radios, where you should add a.disabled class to the label you can use the:disabled attribute.

Messages

Basic

There are four types of styles for messages: default, error, alert and success. All of them can contain links, which will have appropriate colors.

Messages spanning for full width (of the site, usually) should have .full-width class. The text will be centered.

Positioning

Messages can be easily positioned accordingly to correlating fields using following classes: above, below, left, right – where those classes denotate position of the message relatively to the relevant element.

Modals

Regular Modals

All modals are triggered by having a div with a class .modal. Detailed markup for modal usage can be found in the readme.

Modals with no header

Sometimes you need a simple modal without header and just paragraph with buttons. Use the class .no-head. Paddings will be applied automatically.

Overlay

If you need to cover the content of the page to display the overlay (probably most cases) – you will need an .overlay div appended to body of the page. It will create an semi-transparent black overlay.
