Typography

For custom font usage you have Typekit and Cloud Typography at your disposal. Request passwords from Karolina Szczur or Adam Brault and save in password manager of your choice.

Both of those font services rely on project/kit architecture (read how create a project on Cloud Typography) to which fonts are added. Hoefler&Co introduces the concept of development and production environment – when a project is ready to be published remember about moving to production and uploading font files to relevant static.* AWS buckets (ask Bear for access).

Font families used in majority of &yet's apps are Gotham and Helvetica Neue. When using Gotham be aware of the fact that it's web font served through Cloud Typography.

Buttons

To create a button add button class to <a> or <button>

Disabled buttons

To disable buttons add button-disabled class or for <button> elements feel free to use the disabled attribute.

Buttons come in three different sizes — large, regular (no class) and small.

Modals

Regular Modals

All modals are divs with class modal and need to be triggered by Javascript.

Modals with no header

Sometimes you need a simple modal without header and just paragraph with buttons. Use the class modal-no-head and omit the modal-head container.

Overlay

If you need to cover the content of the page to display the overlay (probably most cases) — you will need an overlay div appended to body of the page or has-modal class added to the body element. Both will create an semi-transparent black overlay.

Progress Bars

All progress bars are built with a meter with an animation attached to it. No calculations of state or value are perfomed (you need to figure this out by yourself).
